# RTX 3080 schema-v2 定向验证

**简体中文** | [English](rtx3080-targeted-validation.en.md)

本报告发布五个 Core 10 候选的定向手工验证。它不是 Agent 自动搜索结果，也不替代不可变的 v1 历史报告。

协议：FP16、默认流、20 次预热、100 次 CUDA Events 采样、五个独立进程 Session、AB/BA 交替顺序、Session spread 最大 5%、中位加速至少 1.01×，且配对 bootstrap 95% 下界大于 1.0。官方与边界输入还要求输出有限、五次运行确定，并且 p99/max 误差不差于 `baseline * 1.10 + 1e-4`。

| 任务 | 结果 | 配对中位加速 | Bootstrap 95% 区间 | 稳定性说明 |
| --- | --- | ---: | ---: | --- |
| 004 | 正式提升 | 182.176× | [179.696×, 185.507×] | 基线/候选 spread 为 2.876%/0.861% |
| 007 | 无法定论 | 仅诊断 11.849× | [11.027×, 12.547×] | 上游基线 spread 13.779%；排除正式结果 |
| 036 | 正式提升 | 56.332× | [56.206×, 56.371×] | 基线/候选 spread 为 0.506%/0.344% |
| 040 | 正式提升 | 23.093× | [23.048×, 23.368×] | 基线/候选 spread 为 1.686%/1.598% |
| 095 | 探索中 | — | — | 正确性通过；尚无五 Session 确认 |

五个定向候选均通过官方与新增边界正确性、有限性、确定性和误差回归检查。原始 suite summary 生成于最终 `inconclusive` 状态补丁之前；规范[结果 JSON](../results/rtx3080_targeted_validation.json)记录了用于本次分类的 Session 中位数、门禁结果以及源码/Driver/原始文件哈希。

本地 NCU 仍因 `ERR_NVGPUCTRPERM` 不可用，因此结果明确标记为 `events_only`。本报告不声称已完成基于硬件计数器的 profile-guided 归因或完整 Agent 闭环。
